
> **L0** is the **primary operational state of a PCIe link as the normal operational state where data packets like TLPs, DLLPs, and Logical Idles are sent and received**. Once the L0 state is reached, the root complex and endpoints can successfully communicate between each other. Alternatively, the PCIe link could transition into a number of low-power states or into another link training state called Recovery.

When the LTSSM reaches L0:

```
Detect        ✓
Polling       ✓
Configuration ✓
L0            ✓
```

the PCIe link is considered **Link Up** and normal traffic can flow.

# What Happens When Entering L0?

After Configuration completes:

```
Configuration.Complete
        |
        v
Configuration.Idle
        |
        v
L0
```

Both sides have already agreed on:

- Link width (x1/x2/x4/x8/x16)
- Lane numbering
- Lane polarity
- Basic PHY synchronization

At this point:

```
Bit Lock      = Yes
Symbol Lock   = Yes
Lane Sync     = Yes
Link Width    = Known
```

The link is ready for real PCIe traffic.

# What Traffic Flows in L0?

Unlike Polling and Configuration, which exchange TS1/TS2 training sequences, L0 carries actual PCIe packets.

```
Root Complex                     NVMe SSD

Memory Write TLP -------------->
Memory Read TLP  -------------->
                     <---------- Completion TLP

ACK DLLP          <------------>
Flow Control DLLP <------------>
```

Normal traffic includes:

### TLPs (Transaction Layer Packets)

Examples:

```
Memory Read
Memory Write
Configuration Read
Configuration Write
Messages
Completions
```

### DLLPs (Data Link Layer Packets)

Examples:

```
ACK
NAK
Flow Control
Power Management
```

# What Happens Immediately After L0?

One of the first things that occurs is **Data Link Layer initialization**, including flow-control credit exchange between the two link partners. The Data Link Layer must initialize before full packet traffic can proceed reliably.

Simplified:

```
PHY Link Up
      |
      v
DLL Initialization
      |
      v
Flow Control Credits Exchanged
      |
      v
Normal TLP Traffic
```

# NVMe Example

Suppose a PC boots with a PCIe Gen4 x4 NVMe SSD.

### Before L0

```
Detect
Polling
Configuration
```

Link training completes.

### Enter L0

```
Speed = Gen1 initially
Width = x4
```

Now the OS can enumerate the SSD.

Example transactions:

```
Config Read
Config Write
BAR Assignment
NVMe Register Access
```

Then:

```
Create Admin Queue
Identify Controller
Get Features
```

All of these occur in L0.

# Is the Link Already Running at Gen4/Gen5?

Not necessarily.

A common misconception is:

```
L0 = Final Speed
```

Not always.

Typically:

```
Training starts at Gen1
(2.5 GT/s)
```

The link enters L0 at that speed first.

If both devices support higher speeds:

```
L0
 |
 v
Recovery
 |
 v
Speed Change
 |
 v
L0
```

The link may retrain to:

```
Gen2
Gen3
Gen4
Gen5
```

using the Recovery state.

# What Can Cause Exit from L0?

The link doesn't stay in L0 forever.

Events can force transitions:

### 1. Speed Change

```
L0
 |
 v
Recovery
 |
 v
L0
```

Example:

```
Gen1 → Gen4
```

### 2. Error Recovery

If symbol lock or block lock is lost:

```
L0
 |
 v
Recovery
```

The PHY retrains and attempts to restore the link.

### 3. Power Management

```
L0
 |
 +--> L0s
 |
 +--> L1
 |
 +--> L2
```

Used for power savings.

# What Does an NVMe Engineer Usually See?

A healthy SSD typically shows:

```
LTSSM = L0
Speed = Gen4
Width = x4
```

This means:

```
Link Training Completed
DLL Active
TLPs Flowing
NVMe Commands Working
```

If LTSSM repeatedly moves:

```
L0
 |
 v
Recovery
 |
 v
L0
```

that often indicates:

- Signal integrity issues
- Equalization problems
- Marginal PHY settings
- Connector issues
- Excessive bit errors

# For an NVMe SSD Specifically

The first NVMe-related activities after L0 are typically:

```
1. PCIe Enumeration
2. BAR Mapping
3. Read CAP Register
4. Program AQA
5. Program ASQ
6. Program ACQ
7. Set CC.EN = 1
8. Wait for CSTS.RDY = 1
9. Create I/O Queues
10. Send NVMe Commands
```

None of this can happen until the LTSSM reaches **L0**, because L0 is the first state where normal PCIe packet transmission is allowed.
